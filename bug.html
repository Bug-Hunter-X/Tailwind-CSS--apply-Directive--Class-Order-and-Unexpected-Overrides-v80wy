I encountered an unexpected behavior while using Tailwind CSS's `@apply` directive within a component. I was applying a class containing multiple utility classes, and some of them seemed to be ignored or overridden unexpectedly. For instance: ```html
This text should be gray, centered, large, and bold.
``` The `text-center`, `text-xl`, and `font-bold` classes worked fine when applied directly, but when using `@apply`, only some were applied. Inspecting the browser's developer tools revealed that the order of classes within the `@apply` directive is critical and certain classes were being overridden silently, leading to unpredictable results.
